# CLAUDE_WORKFLOW.md

## Purpose

This document defines the standard workflow Claude must follow when working on this project.

This workflow should be followed before every development task.

---

# Phase 1

Read documentation.

Always begin by reading:

dashboard-bible.md

PROJECT.md

DESIGN.md

STYLE_GUIDE.md

CODING_RULES.md

TODO.md

Then read any documentation directly related to the requested feature.

Never skip this step.

---

# Phase 2

Understand the request.

Do not immediately write code.

Instead:

Inspect existing files.

Understand architecture.

Locate reusable components.

Identify relevant data.

Plan the implementation.

---

# Phase 3

Planning

Produce a short implementation plan.

List the files that will be edited.

Avoid unnecessary edits.

---

# Phase 4

Development

Modify only the required files.

Never rewrite the project.

Reuse components.

Reuse styles.

Reuse data.

Keep code clean.

---

# Phase 5

Verification

Check:

Broken imports

Broken paths

Console errors

Responsive layout

Navigation

Data flow

Vercel compatibility

Performance

---

# Phase 6

Documentation

If necessary update:

ROADMAP

TODO

CHANGELOG

Documentation should always reflect the current project.

---

# Phase 7

Completion

Provide only:

Files changed

Summary

Warnings

Suggested commit message

Do not provide long explanations unless requested.

---

# Engineering Principles

Prefer:

Simple

Reusable

Maintainable

Readable

Scalable

Efficient

Professional

---

# Never

Never redesign the UI.

Never overwrite unrelated code.

Never remove working features.

Never duplicate logic.

Never create inconsistent styling.

Never ignore existing documentation.

---

# Final Rule

Work as if you are a senior software engineer maintaining a long-term production application.

Quality is more important than speed.

Consistency is more important than creativity.

Long-term maintainability is more important than short-term convenience.
