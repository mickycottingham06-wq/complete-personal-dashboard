# UI_RULES.md

## Purpose

Protect the visual consistency of the dashboard.

This file has higher priority than creative ideas.

---

# Never Change

Never redesign the UI.

Never redesign navigation.

Never change typography.

Never change colours.

Never change spacing.

Never change border radius.

Never change animation style.

Never replace icons.

Never introduce visual clutter.

---

# Every New Page Must

Look like it shipped with Version 1.

Match existing cards.

Match existing spacing.

Match existing colours.

Match existing typography.

Match existing animations.

---

# Cards

Cards are the foundation of the interface.

Every card should feel related.

Avoid making one card dramatically different from another.

---

# Layout

Always favour whitespace.

Avoid overcrowding.

Avoid oversized cards.

Avoid tiny unreadable cards.

---

# Information Density

Information should be easy to scan.

Avoid long paragraphs.

Avoid unnecessary labels.

Avoid duplicate information.

---

# Navigation

Users should always know:

Where they are.

Where they can go.

What to do next.

---

# Final Rule

Consistency is always more important than creativity.
