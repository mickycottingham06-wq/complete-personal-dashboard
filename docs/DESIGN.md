# DESIGN.md

## Purpose

This document defines the visual identity of the dashboard.

This design language is considered COMPLETE.

Future development should EXPAND this design, never replace it.

The objective is consistency.

Every new feature should look like it has always been part of the dashboard.

---

# Design Philosophy

The dashboard should feel like:

- Apple's internal dashboard
- A luxury supercar digital cockpit
- A premium trading terminal
- A modern operating system

The interface should communicate:

- Calm
- Precision
- Discipline
- Performance
- Professionalism

The dashboard should never feel:

- Childish
- Cartoonish
- Cluttered
- Cheap
- Busy
- Overdesigned

---

# Core Principles

Every screen should be:

Simple

Readable

Fast

Elegant

Minimal

Useful

The dashboard exists to reduce mental load.

Not increase it.

---

# Colour Palette

Always use the existing dashboard colours.

Never redesign the colour scheme.

Use colours consistently.

Dark background.

Premium card colours.

Soft blue/cyan highlights.

Green only for positive values.

Red only for warnings or losses.

Orange only for streaks or attention.

Never introduce random colours.

---

# Background

Keep the current dark background.

Use subtle gradients only if they already match the project.

Avoid visual noise.

---

# Cards

Cards are the primary UI element.

Every card should:

Have rounded corners.

Use the current border radius.

Use the existing glass effect.

Use subtle shadows.

Have consistent padding.

Use the same spacing as existing cards.

Cards should never feel oversized.

Cards should never feel cramped.

---

# Typography

Keep the existing fonts.

Titles should be bold.

Values should stand out.

Supporting text should be smaller.

Avoid paragraphs inside cards.

Dashboard information should be readable within seconds.

---

# Icons

Use one consistent icon style.

Never mix multiple icon libraries.

Icons should support the information.

Never dominate it.

---

# Navigation

Keep the existing navigation.

Do not redesign navigation.

Do not move navigation without being asked.

Every page should feel connected.

---

# Layout

The layout should breathe.

Use whitespace.

Do not fill every gap.

Avoid unnecessary widgets.

Less is more.

---

# Animations

Animations should be subtle.

Allowed:

Hover lift

Fade

Glow

Smooth transitions

Progress animations

Not allowed:

Large bounces

Spinning effects

Flashing

Distracting movement

---

# Widgets

Widgets should match existing cards.

Weather.

Calendar.

Bills.

Quote.

Search.

All should use identical spacing and styling.

---

# Progress Bars

Use clean premium bars.

Rounded.

Smooth animations.

Consistent colours.

Never oversized.

---

# Charts

Charts should be simple.

Minimal labels.

Minimal grid lines.

No bright colours.

Only show useful information.

---

# Forms

Forms should be minimal.

Simple labels.

Clean inputs.

Consistent spacing.

---

# Mobile

The mobile version is equally important.

Cards should stack naturally.

Nothing should overflow.

Navigation should remain easy to reach.

---

# Consistency Rules

Before creating anything new ask:

Does this look like it belongs?

Could this card exist on every page?

Does it match existing spacing?

Does it use existing colours?

Could someone tell it was added later?

If the answer is yes, redesign it until it blends in.

---

# Never Do These Things

Never redesign the dashboard.

Never replace existing cards.

Never introduce inconsistent spacing.

Never randomly change colours.

Never change fonts.

Never use different border radii.

Never replace icons without reason.

Never add unnecessary decoration.

Never create visual clutter.

Never sacrifice simplicity.

---

# Final Rule

The dashboard should always feel like one product built by one designer.

Not a collection of different ideas.

Every future feature must inherit this design language.
