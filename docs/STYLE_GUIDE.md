# STYLE_GUIDE.md

## Purpose

This document defines the exact styling standards for every new feature added to the dashboard.

Claude must follow these standards before creating any HTML, CSS or JavaScript.

If unsure, copy the style of existing components rather than inventing something new.

---

# General Rule

Never create a component that looks different from the rest of the dashboard.

Every new feature should feel like it shipped with Version 1.

---

# Visual Style

Style Keywords

- Premium
- Minimal
- Modern
- Professional
- Glassmorphism
- Clean
- Elegant
- High-performance
- Apple inspired
- Tesla inspired

Avoid:

- Cartoon styling
- Bright colours
- Thick borders
- Heavy shadows
- Visual clutter

---

# Card Standards

All cards should:

Use existing border radius.

Use the existing glass effect.

Use the existing background colour.

Use consistent padding.

Use the existing shadow.

Use the same hover animation.

Never create oversized cards.

Never create tiny unreadable cards.

Cards should always align cleanly.

---

# Spacing

Maintain consistent spacing everywhere.

Use the same spacing values already found throughout the dashboard.

Avoid random margins.

Avoid inconsistent gaps.

Whitespace should feel intentional.

---

# Typography

Titles

Large

Bold

Simple

Body Text

Medium weight

Easy to scan

Supporting Text

Small

Grey

Never use multiple font families.

---

# Colours

Reuse existing colours only.

Primary

Current dashboard accent colour.

Success

Green.

Warning

Orange.

Danger

Red.

Neutral

Grey.

Never invent new accent colours.

---

# Icons

Use the same icon library already used by the project.

Icons should:

Support information.

Not dominate it.

Remain consistent in size.

---

# Buttons

Buttons should:

Match existing style.

Have subtle hover animations.

Have consistent padding.

Use rounded corners.

Never create oversized buttons.

---

# Progress Bars

Always rounded.

Smooth animation.

Consistent thickness.

Never excessively tall.

Always display percentage when appropriate.

---

# Charts

Charts should:

Match dashboard colours.

Minimal grid lines.

Minimal labels.

No unnecessary decoration.

Easy to read.

---

# Tables

Tables should:

Use existing styling.

Alternating row shading only if already present.

Clean spacing.

Readable on mobile.

---

# Widgets

Every widget should follow the exact same card styling.

Weather.

Calendar.

Bills.

Search.

Quotes.

AI Assistant.

Business.

Finance.

Goals.

Everything should feel like one system.

---

# Hover Behaviour

Cards

Slight lift.

Soft shadow increase.

Subtle glow.

Buttons

Slight brightness.

Pointer cursor.

Nothing excessive.

---

# Animations

Allowed

Fade

Slide

Progress animation

Hover lift

Glow

Not Allowed

Bounce

Spin

Flash

Shake

Large movement

---

# Responsive Design

Desktop first.

Tablet second.

Mobile third.

Never allow horizontal scrolling.

Cards should stack naturally.

Navigation should remain usable.

---

# Accessibility

Maintain readable contrast.

Large clickable areas.

Readable font sizes.

Logical navigation order.

---

# Code Style

Keep HTML clean.

Keep CSS modular.

Keep JavaScript readable.

Avoid duplicate code.

Prefer reusable classes.

Reuse existing utility classes whenever possible.

---

# Before Creating Anything

Ask:

Does this match existing cards?

Does this match spacing?

Does this match colours?

Does this match typography?

Does this match hover animations?

Could somebody tell this was added later?

If yes, redesign until it blends perfectly.

---

# Final Rule

Consistency is more important than creativity.

When in doubt:

Copy the existing design language.

Do not invent a new one.
