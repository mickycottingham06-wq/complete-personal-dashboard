# FEATURE_REQUESTS.md

## Purpose

This document is the master backlog for future dashboard features.

It is intentionally separate from TODO.md.

TODO contains work that has been planned.

FEATURE_REQUESTS contains ideas that may become future features.

Nothing should be built directly from memory.

Every feature idea should first be written here.

---

# Feature Evaluation

Before adding a feature ask:

Does it make money?

Does it save time?

Does it improve health?

Does it improve discipline?

Does it reduce mental load?

Does it provide useful information?

If the answer is no, reconsider adding it.
